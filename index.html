Another implementation of JavaScript? No, much much worse:

{% highlight javascript %} function flipperize(xs) = xs >> where(? is not null) >> map(x => if(x mod 2 == 0) "flip" else "flop"); {% endhighlight %}

That, and tons of meta-programming await you in this dynamically typed, somewhat object-oriented1, interpreted abomination of a scripting language.

Footnotes

  1. Yes, that's a thing. Prexonite Script allows you to consume objects, methods, properties, index accessors, etc. but there is no way to define classes or even prototypes, like with JavaScript. Having said that, there are ways to define custom constructors that return "objects", however, looking for .prototype will be in vain.
